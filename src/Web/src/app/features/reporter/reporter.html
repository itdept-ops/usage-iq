2Get the reporter

The reporter is a small .NET 9 console app in the repo (src/Reporter). Build a release binary, or run it straight from source — on the machine that has the Claude Code / Codex logs.

git clone https://github.com/itdept-ops/usage-iq && cd usage-iq dotnet build src/Reporter -c Release
3Run it

Point it at this server with the key from step 1. By default it watches the logs and pushes new usage continuously; pass --once for a single pass (e.g. from cron).

Keep it running

Run the watcher under your OS service manager so it survives reboots, or schedule --once on a timer. Config can also come from REPORTER_* env vars or an appsettings.json next to the executable.

terminal Linux (systemd)
[Service] Environment=REPORTER_URL={{ serverUrl() }} Environment=REPORTER_KEY=uiq_… ExecStart=/opt/usage-iq/usage-iq-reporter Restart=always
schedule Windows (Task Scheduler)
schtasks /create /tn "UsageIQ Reporter" /sc onlogon ^ /tr "usage-iq-reporter.exe --url {{ serverUrl() }} --key uiq_…"
event_repeat cron (hourly --once)
0 * * * * usage-iq-reporter \ --url {{ serverUrl() }} --key uiq_… --once
